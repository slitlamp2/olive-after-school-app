# 올리브청소년방과후 센터 활동앱 — 현재 요구사항 컨텍스트

## 1. 앱 개요

- **앱 이름**: `올리브청소년방과후 센터 활동앱`
- **주 사용자**: 초등~고등 연령의 발달장애 청소년을 방과후에 돌보는 사회복지사
- **운영 시간**: 오후 2시 ~ 7시
- **주 목적**: 사진과 입력 정보를 바탕으로 `일일활동일지`, `품위서`, `계획서`를 자동 생성하고 최종 결과를 모두 **HWP(.hwp)** 로 저장

---

## 2. 메인 화면 요구사항

- 메인 제목은 `올리브청소년방과후 센터 활동앱`
- 메인 화면에는 **버튼 4개**가 동일한 계층으로 배치되어야 함
- 메인 화면의 부제 `사회복지사를 위한 활동 기록 자동화 도구`는 표시하지 않음

| 순서 | 버튼 제목 | 설명 |
|------|-----------|------|
| 1 | 일일활동일지생성 | 사진 3장과 사진별 메타정보를 바탕으로 활동일지 생성 |
| 2 | 품위서 | 영수증 사진을 바탕으로 품위서 생성 |
| 3 | 계획서 | 품위서 내용 또는 수기 입력 + 사진으로 계획서 생성 |
| 4 | 기타 | 향후 확장 기능용 placeholder |

---

## 3. 공통 UI 요구사항

- 각 화면의 `← 메인으로` 버튼 글씨는 기본보다 크게 표시
- 사진 업로드는 드래그 앤 드롭이 아니라 **버튼형 업로드 UI** 사용
- 생성 완료 후 저장 경로를 표시하고, 사용자가 **클릭해서 HWP 파일을 직접 열 수 있어야 함**
- OpenAI API 키는 반드시 `app/.env` 에서 불러와야 하며, 유효하지 않거나 비어 있으면 데모 모드가 아니라 오류/안내가 명확히 보여야 함

---

## 4. 문서 생성 공통 원칙

- 최종 산출물은 모두 **HWP(.hwp)** 형식
- 템플릿의 **표 구조와 서식은 유지하고 내용만 채움**
- 템플릿 경로
  - `D:\일일활동일지양식.hwp`
  - `D:\품위서양식.hwp`
  - `D:\계획서양식.hwp`
- 출력 파일은 앱의 `outputs` 폴더에 새 파일명으로 저장
- 가능하면 생성 후 웹에서 경로를 표시하고 `파일 열기` 버튼으로 바로 열 수 있어야 함

## 5. 일일활동일지생성 요구사항

### 5-1. 입력 UI

- 기본 정보에는 아래 두 항목만 사용
  - `활동 명단`
  - `활동 날짜`
- 사진 업로드는 **총 3개 버튼**
- 각 사진마다 아래 입력칸이 각각 따로 존재해야 함
  - `활동 시간`
  - `활동 장소`
  - `사진별 특이사항`

### 5-2. 문서 작성 기준

- 사진 분석 결과와 입력 정보를 바탕으로 당일 활동 내용을 자동 생성
- 생성된 텍스트는 템플릿의 **`프로그램 내용`이 아니라 `프로그램 활동내역` 칸**에 들어가야 함
- 내용은 시간대 제목 없이 **본문만** 들어가야 함
  - 예: `[14:30~15:00 등원 및 자유활동]` 같은 헤더는 출력하지 않음
- `없음`, `특이사항 없음`, `이상 없음`, 형식적인 마무리 문장은 출력하지 않음
- 실제 활동이 보이지 않는 시간대는 빈 값으로 두거나 생략
- 생성 문체는 **선생님이 보호자에게 오늘 아이 활동을 설명하듯 따뜻하고 쉬운 말투**
- 공문서 문체(`~하였다`, `~실시하였다`)는 지양
- 아동 이름은 항목 첫 문장에서만 자연스럽게 사용하고, 이후 문장은 반복을 줄임
- 이름 뒤 조사는 가능하면 `은/는` 형태를 우선 사용

### 5-3. 길이와 형식

- 각 활동 항목은 사진 근거 중심으로 충분히 구체적으로 작성
- 한 줄짜리 짧은 문장보다, 보호자가 상황을 떠올릴 수 있게 표정·행동·상호작용·사용 교구를 포함
- 생성 내용의 불필요한 형식 문장 예시
  - `이상이 없습니다`
  - `정상적으로 완료되었습니다`
  - `특이사항: 이용자 돌발행동 및 대처방법 없음`
- 위와 같은 문장은 출력하지 않음

### 5-4. 사진 삽입 요구사항

- 문서 하단 `활 동 사 진` 아래의 **좌측 셀에는 사진 1**
- 같은 행의 **우측 셀에는 사진 2**
- 사진은 업로드된 사진 중에서 넣기 적절한 사진을 사용
- 현재 사진 자동 삽입은 안정화가 필요한 영역이며, 최종 목표는 위 두 칸에 맞춰 자동으로 들어가도록 하는 것

### 5-5. HWP 표시 형식

- 생성 내용은 첫 줄을 비워서 시작
- 생성 내용 글씨 크기는 **9.5pt**

## 6. 품위서 요구사항

### 6-1. 입력 UI

- 영수증 업로드 버튼은 **3개**
- 영수증 사진 1장 이상 업로드 가능

### 6-2. 처리

- OpenAI로 영수증 사진을 분석하여 다음 항목 추출
  - 구입일자
  - 매장명
  - 품명
  - 수량
  - 단위
  - 단가
  - 금액
  - 비고
- 추출 결과를 품위서 템플릿의 해당 셀에 매핑

### 6-3. 출력

- 결과는 `품위서.hwp` 계열 파일로 저장
- 생성 후 필요 시 계획서 생성 단계로 연결 가능

## 7. 계획서 요구사항

### 7-1. 입력 UI

- 사진 업로드 버튼은 **6개**
- 품위서 내용을 이어받거나 직접 입력 가능

### 7-2. 생성 내용

- OpenAI가 아래 항목을 생성
  - 목적
  - 목표
  - 프로그램 진행 내용
  - 기대 효과

### 7-3. 출력

- 계획서 템플릿에 내용을 채워 **HWP(.hwp)** 로 저장
- 사진은 템플릿의 지정 영역 또는 다음 페이지에 배치할 수 있음

## 8. 기술 구성

- **백엔드**: Flask
- **프론트엔드**: HTML, CSS, JavaScript
- **AI**: OpenAI Vision / 텍스트 생성
- **문서 자동화**: HWP COM 자동화 기반 서비스
- **환경 변수**: `python-dotenv`
- **파일 열기**: `os.startfile()`

---

## 9. 현재 구현 시 주의사항

- HWP 자동화는 COM 기반이라 단일 스레드 잠금과 COM 초기화/해제가 중요함
- HWP 보안 허용 팝업은 가능하면 사용자 개입 없이 처리되어야 함
- 사진 자동 삽입 기능은 가장 불안정한 영역이므로, 앱 전체 구동 안정성을 해치지 않도록 분리해서 다뤄야 함
- 앱이 구동되지 않을 때는 먼저 서버 응답 여부와 HWP 자동화 루틴 블로킹 여부를 확인

## 10. 핵심 플로우 요약

- **일일활동일지**
  - 활동 명단/날짜 입력
  - 사진 3장 + 사진별 시간/장소/특이사항 입력
  - OpenAI 분석
  - `프로그램 활동내역` 칸에 본문 입력
  - 하단 활동사진 칸에 좌/우 사진 배치
  - `.hwp` 저장 후 바로 열기 가능

- **품위서**
  - 영수증 사진 업로드
  - OpenAI OCR/구조화
  - 품위서 템플릿 채우기
  - `.hwp` 저장

- **계획서**
  - 품위서 정보 또는 직접 입력
  - 사진 업로드
  - 목적/목표/프로그램 내용/기대효과 생성
  - 계획서 템플릿 채우기
  - `.hwp` 저장
