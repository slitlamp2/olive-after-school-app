# 올리브청소년방과후 센터 활동앱 — 현재 요구사항 컨텍스트

## 1. 앱 개요

- **앱 이름**: `올리브청소년방과후 센터 활동앱`
- **주 사용자**: 초등~고등 연령의 발달장애 청소년을 방과후에 돌보는 사회복지사
- **운영 시간**: 오후 2시 ~ 7시
- **주 목적**: 사진과 입력 정보를 바탕으로 `일일활동일지`, `품의서`, `계획서`를 자동 생성하고 최종 결과를 모두 **HWP(.hwp)** 로 저장
- **서버 포트**: `5006` (Flask, `0.0.0.0`)
- **ngrok 지원**: `USE_NGROK=1` + `NGROK_AUTHTOKEN` 설정 시 외부 터널 자동 연결

## 2. 메인 화면 요구사항

- 메인 제목은 `올리브청소년방과후 센터 활동앱`
- 메인 화면에는 **버튼 3개**가 동일한 계층으로 배치

| 순서 | 버튼 제목 | 설명 |

| 2 | 품의서 | 영수증 사진 업로드 후 품의서를 자동으로 생성합니다. |
| 3 | 계획서 | 품의서 내용과 사진으로 목적·목표·기대효과를 생성합니다. |

## 3. 공통 UI 요구사항

- 각 화면의 `← 메인으로` 버튼 글씨는 기본보다 크게 표시
- 사진 업로드는 드래그 앤 드롭이 아니라 **버튼형 업로드 UI** 사용
- 생성 완료 후 저장 경로를 표시하고, 사용자가 **클릭해서 HWP 파일을 직접 열 수 있어야 함**
- OpenAI API 키는 반드시 `app/.env` 에서 불러와야 하며, 유효하지 않거나 비어 있으면 오류/안내가 명확히 보여야 함

## 4. 문서 생성 공통 원칙

- 최종 산출물은 모두 **HWP(.hwp)** 형식
- 템플릿의 **표 구조와 서식은 유지하고 내용만 채움**
- 템플릿 경로
  - `D:\일일활동일지양식.hwp`
  - `D:\품위서양식.hwp`
  - `D:\계획서양식.hwp`
- 출력 파일은 앱의 `outputs` 폴더에 새 파일명으로 저장
- 가능하면 생성 후 웹에서 경로를 표시하고 `파일 열기` 버튼으로 바로 열 수 있어야 함
- HWP COM 종료 시 `SetMessageBoxMode(0x00020000)` 으로 저장 대화상자 자동 "아니오" 처리
- 사진 삽입 워커 실행 전 1.5초 대기 → 한글 이중 실행 방지

## 5. 일일활동일지생성 요구사항

### 5-1. 입력 UI

- 기본 정보에는 아래 두 항목만 사용
  - `활동 명단`
  - `활동 날짜`
- 사진 업로드는 **총 3개 버튼**
- 각 사진마다 아래 입력칸이 각각 따로 존재해야 함 (위→아래 순서)
  - `활동 시간` → HWP 양식의 `시간1` / `시간2` / `시간3` 에 매핑
  - `프로그램 내용` → HWP 양식의 `활동1` / `활동2` / `활동3` 에 매핑
  - `활동 장소`
  - `특이사항`

### 5-2. HWP 슬롯 매핑 (시간·활동)

- 사진 1의 활동 시간 → `시간1`, 프로그램 내용 → `활동1`
- 사진 2의 활동 시간 → `시간2`, 프로그램 내용 → `활동2`
- 사진 3의 활동 시간 → `시간3`, 프로그램 내용 → `활동3`
- 업로드 장수보다 큰 슬롯은 빈 문자열로 치환(플레이스홀더 제거)
- `_fill_purchase_placeholder_cell` (replace_text + fill_cell_next_to_label 폴백) 재사용

### 5-3. 헤더 채우기

- `일시` ← 날짜만 (시간은 넣지 않음)
- `이용시간` ← 빈 문자열 (월별 이용시간에는 자동 입력하지 않음)
- `이용자` ← 활동 명단

### 5-4. 문서 작성 기준

- 사진 분석 결과와 입력 정보를 바탕으로 당일 활동 내용을 자동 생성
- 프로그램 내용(직접입력)이 있으면 AI가 해당 사진의 참여 서술에 반드시 반영
- 생성된 텍스트는 템플릿의 **`프로그램 활동내역` 칸**에 들어가야 함
- 내용은 시간대 제목 없이 **본문만** 들어가야 함
  - 예: `[14:30~15:00 등원 및 자유활동]` 같은 헤더는 출력하지 않음
- `없음`, `특이사항 없음`, `이상 없음`, 형식적인 마무리 문장은 출력하지 않음
- 실제 활동이 보이지 않는 시간대는 빈 값으로 두거나 생략
- 문체는 사회복지사가 관찰 내용을 보고서에 작성하듯 기술
- 문장 끝은 `~했음`, `~하였음`, `~보였음` 등 보고서 종결어미로 마무리
- 아동 이름은 항목 첫 문장에서만 자연스럽게 사용하고, 이후 문장은 반복을 줄임

### 5-5. 길이와 형식

- 각 활동 항목은 사진 근거 중심으로 충분히 구체적으로 작성
- 한 줄짜리 짧은 문장보다, 보호자가 상황을 떠올릴 수 있게 표정·행동·상호작용·사용 교구를 포함
- 불필요한 형식 문장(`이상이 없습니다`, `정상적으로 완료되었습니다` 등)은 출력하지 않음

### 5-6. 사진 삽입 요구사항

- 문서 하단 `활 동 사 진` 아래의 **좌측 셀에는 사진 1** (플레이스홀더: `사진1입력`)
- 같은 행의 **우측 셀에는 사진 2** (플레이스홀더: `사진2입력`)
- 삽입 시 앵커 치환 방식으로 플레이스홀더 문구(`사진1입력`, `사진2입력`)를 **완전히 제거**
- EXIF Orientation 보정: `ImageOps.exif_transpose` 로 휴대폰 사진 방향 자동 맞춤
- 사진은 업로드된 사진 중에서 넣기 적절한 사진을 사용

### 5-7. HWP 표시 형식

- 생성 내용은 첫 줄을 비워서 시작
- 생성 내용 글씨 크기는 **9.5pt**

## 6. 품의서 요구사항

### 6-1. 입력 UI

- 영수증 업로드 버튼은 **3개** (영수증 1 / 영수증 2(선택) / 영수증 3(선택))
- 영수증 사진 1장 이상 업로드 필수
- 각 영수증마다 **🔄 회전** / **↔ 뒤집기** 버튼 제공
  - 미리보기에 CSS transform 으로 즉시 반영
  - 서버 전송 시 `receipt{N}_rotate` (0/90/180/270) · `receipt{N}_flip` (0/1) 값 포함
  - Pillow로 실제 픽셀 변환 후 OpenAI 분석·HWP 삽입에 사용

### 6-2. 처리

- 영수증별 개별 OpenAI API 호출 → Python에서 합산·병합
- 각 영수증에서 추출 항목
  - 구입일자, 매장명, 품명, 수량, 단위, 단가, 금액, 비고
- 합산 규칙: 합계는 각 영수증 하단 합계를 정수 합산, 날짜는 가장 최근, 매장명은 `|` 구분 병합

### 6-3. HWP 슬롯 매핑

- **장별 개별 필드**
  - 영수증 1 합계 → `영수증1합계`, 거래처 → `영수증1거래처`
  - 영수증 2 합계 → `영수증2합계`, 거래처 → `영수증2거래처`
  - 영수증 3 합계 → `영수증3합계`, 거래처 → `영수증3거래처`
- **글로벌 거래처** ← 첫 번째 영수증의 매장명
- **품목 표** ← 전체 품목 리스트를 `품명` 헤더 기준 행으로 채움

### 6-4. 영수증 사진 삽입

- 플레이스홀더: `영수증1사진` / `영수증2사진`(또는 `영수증사진2`) / `영수증3사진`(또는 `영수증사진3`)
- 삽입 시 앵커 치환 방식으로 플레이스홀더 문구 **완전 제거**
- EXIF Orientation 보정 적용
- 구형 양식 폴백: `영수증사진` · `영수증입력` 등 (1장만)

### 6-5. 출력

- 결과는 `품의서_YYYYMMDD_HHMMSS.hwp` 파일로 저장
- 생성 후 계획서 생성 단계로 연결 가능
- 세션에 품의서 데이터(품목 리스트·합계 등) 저장 → 계획서에서 자동 불러오기

## 8. 기술 구성

- **백엔드**: Flask (포트 5006, `threaded=False`)
- **프론트엔드**: HTML, CSS, JavaScript
- **AI**: OpenAI Vision / 텍스트 생성
- **문서 자동화**: HWP COM 자동화 기반 서비스 (`hwp_service.py`)
- **사진 삽입**: 별도 프로세스 (`photo_insert_worker.py`, 클립보드 BMP + Ctrl+V)
- **이미지 전처리**: Pillow (`ImageOps.exif_transpose`, 회전·뒤집기)
- **환경 변수**: `python-dotenv` (`app/.env`, UTF-8, 실행 위치 무관)
- **네트워크 터널**: `pyngrok` (선택)
- **파일 열기**: `os.startfile()`
- **버전 관리**: Git → GitHub (`slitlamp2/olive-after-school-app`)

## 9. 현재 구현 시 주의사항

- HWP 자동화는 COM 기반이라 단일 스레드 잠금(`HWP_LOCK`)과 COM 초기화/해제가 중요함
- HWP 보안 허용 팝업은 `RegisterModule` + `SetMessageBoxMode`로 사용자 개입 없이 처리
- HWP 종료 시 `SetMessageBoxMode(0x00020000)` → 저장 대화상자 자동 "아니오"
- 사진 삽입은 별도 프로세스에서 비동기 처리 → 앱 전체 구동 안정성 유지
- 문서 생성 후 사진 워커 시작 전 **1.5초 대기** → 한글 이중 실행 방지
- 앱이 구동되지 않을 때는 먼저 서버 응답 여부와 HWP 자동화 루틴 블로킹 여부를 확인

## 10. 핵심 플로우 요약

- **일일활동일지**
  - 활동 명단/날짜 입력
  - 사진 3장 + 사진별 활동시간/프로그램내용/장소/특이사항 입력
  - OpenAI 분석 (프로그램 내용 직접입력 시 AI 반영)
  - HWP 양식의 `시간1~3`, `활동1~3` 플레이스홀더에 자동 채움
  - `프로그램 활동내역` 칸에 본문 입력
  - 하단 `사진1입력`/`사진2입력` 칸에 사진 배치 (플레이스홀더 제거)
  - `.hwp` 저장 후 바로 열기 가능

- **품의서**
  - 영수증 사진 업로드 (최대 3장, 회전·뒤집기 지원)
  - 영수증별 개별 OpenAI OCR → Python 합산·병합
  - 장별 합계·거래처 + 전체 품목표 채우기
  - 영수증 사진을 `영수증1사진`~`영수증3사진` 슬롯에 삽입 (플레이스홀더 제거)
  - `.hwp` 저장 → 계획서 연결 가능

- **계획서**
  - 품의서 정보 또는 직접 입력
  - 사진 업로드 (최대 6장)
  - 목적/목표/프로그램 내용/기대효과 생성
  - 계획서 템플릿 채우기
  - `.hwp` 저장
